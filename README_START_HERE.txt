VetBara LAN Bootstrap Installer v55

==================================

macOS:
- Double-click: Start VetBara Admin - macOS.command
- Or: Start VetBara Centre - macOS.command

Windows:
- Double-click: Start VetBara Admin - Windows.bat
- Or: Start VetBara Centre - Windows.bat

The Start scripts now automatically install the local Node.js runtime on first run when it is missing.
Internet is required only for this first runtime installation. The downloaded Node.js archive is verified using SHA256 from the official Node.js SHASUMS256.txt file.

After the runtime has been installed, VetBara runs locally over LAN without internet.

If macOS blocks the command because it comes from an unidentified developer:
1. Open System Settings > Privacy & Security.
2. Click Open Anyway for the blocked command.
Or run:
   xattr -dr com.apple.quarantine /path/to/VetBara_LAN_Bootstrap_Installer


Version v51 note: default port is 3010 to avoid old browser/service-worker cache from previous VetBara test runs. If you see a blank page, open http://127.0.0.1:3010/__reset.html and then restart.


V53 note:
- Admin runner no longer forces every URL back to role=Admin.
- Centre access links generated by Admin keep role=Centre and can be copied into the Centre runner.
- In split Admin/Centre LAN operation, use the Centre runner for the Centre workspace; the Admin link host may change after Wi-Fi/router changes, but the runner keeps id/token and rebuilds the current LAN URL.


v55 note: QR links for tablets/candidates/examiners use the LAN base URL from /api/health when the local admin/centre page is opened on 127.0.0.1 or localhost.

HTTPS / GPS mode
----------------
VetBara will automatically switch from HTTP to HTTPS when it finds a certificate pair in app/certs/.
Use this for tablet GPS in LAN mode.

Expected filenames:
  app/certs/vetbara-lan.pem
  app/certs/vetbara-lan-key.pem

Example mkcert command on the computer running VetBara:
  mkdir -p app/certs
  mkcert -cert-file app/certs/vetbara-lan.pem -key-file app/certs/vetbara-lan-key.pem 192.168.0.186 localhost 127.0.0.1

Replace 192.168.0.186 with the current LAN IP of the computer running VetBara.
Install and fully trust the mkcert root CA on every tablet. Without that, browsers may still block GPS.

When HTTPS is active, the terminal prints:
  Protocol: HTTPS
  Use from other LAN devices: https://<LAN-IP>:3010/...
