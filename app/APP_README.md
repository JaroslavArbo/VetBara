# VetBara LAN Portable app

This folder contains the built VetBara web application, local LAN server, and runtime data folder.

Runtime data are stored in `app/data/`:
- `active-test-package.json` - prefilled exam package data
- `sync/events.json` - portable sync/log events
- `final/` - generated final packages
- `scans/` - scan workflow data

The server listens on `0.0.0.0` and opens the LAN IP URL in the default browser.


V53 note:
- Admin runner no longer forces every URL back to role=Admin.
- Centre access links generated by Admin keep role=Centre and can be copied into the Centre runner.
- In split Admin/Centre LAN operation, use the Centre runner for the Centre workspace; the Admin link host may change after Wi-Fi/router changes, but the runner keeps id/token and rebuilds the current LAN URL.

v56 update:
- Field tablet links in Centre Field site setup use the LAN base URL from /api/health, not 127.0.0.1.
- QR and visible tablet link are generated from the same URL value.
- Centre runner no longer redirects field-tablet mode back to the Centre workspace.

## v60 update

Field tablet mode no longer depends on GPS for manual orientation. In the map toolbar, paste coordinates in the form:

49.205828390088236, 16.427684681686493

Then press "Pouzit souradnice" / "Použít souřadnice". The map centers on that position and shows it as the GPS/manual position marker.


## v60 embedded Package 1

This installer includes the uploaded current `.vet` package as the active portable test package. On startup the Admin and Centre endpoints load `app/data/active-test-package.json`, copied from `VetBara-Package1-en-2026-06-13.vet`. The original `.vet` archive is stored in `app/data/packages/`.

## v60 rules translation editor

The Admin authoring screen now uses a dropdown for the rules translation language. For each rules document the English source/original is shown on the left and the editable translation fields are shown on the right.
